# Flujo embebido de Disclosure (iframe)

Esta guía explica cómo ejecutar el flujo de **Disclosure** dentro de un iframe usando el SDK web `@soyio/soyio-widget`.

:::info[Alcance del flujo embebido]
Actualmente, el modo embebido soporta solo Disclosure con `disclosure_request_id` ya creado.

No está soportada la creación del disclosure request desde frontend en este modo.
:::

:::warning[Configuración obligatoria de CSP (`whitelisted_domains`)]
Para usar Disclosure embebido debes registrar los dominios host en `whitelisted_domains` de tu compañía.

Si el dominio donde montas el iframe no está permitido, el navegador bloqueará el render del flujo por política CSP (`frame-ancestors`).

En modo embebido, Soyio arma `frame-ancestors` con:

- `referer` de la request
- `whitelisted_domains` de la configuración
:::

## Cuándo usar embedded vs popup

- Usa **popup** (`SoyioWidget`) cuando quieras un flujo desacoplado de tu layout.
- Usa **embedded** (`EmbeddedSoyioWidget`) cuando quieras mantener el flujo dentro de tu página.

## Requisitos

1. Tener un `disclosure_request_id` ya creado (requisito obligatorio para embedded).
2. Configurar `whitelisted_domains` con los orígenes donde se montará el iframe.
3. Instalar el SDK web:

```bash
npm install @soyio/soyio-widget
```

## Implementación

```html
<div id="embedded-disclosure"></div>

<script>
  import { EmbeddedSoyioWidget } from "@soyio/soyio-widget";

  const widget = new EmbeddedSoyioWidget({
    request: "disclosure",
    configProps: {
      disclosureRequestId: "<disclosure_request_id>",
      customColor: "#0F172A", // opcional
    },
    onReady: () => {
      console.log("Embedded disclosure ready");
    },
    onEvent: (event) => {
      console.log("Embedded disclosure event:", event);
    },
    height: "720px", // opcional
    minHeight: "640px", // opcional
    isSandbox: true, // opcional
  });

  widget.mount("#embedded-disclosure");
</script>
```

## Passkeys en flujo embebido

La creación y autenticación con passkeys no siempre está soportada dentro de iframes/webviews. En esos casos, el SDK abre un popup temporal para completar passkey y luego retorna automáticamente al iframe.

## Troubleshooting CSP

Si ves un iframe en blanco o errores en consola tipo `Refused to frame ... because an ancestor violates Content Security Policy`, revisa:

1. Que el origen host esté en `whitelisted_domains`.
2. Que el dominio de staging/producción sea exactamente el esperado.
