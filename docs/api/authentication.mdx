# Autenticación

Autentica tus requests con [Bearer tokens sobre HTTPS](https://datatracker.ietf.org/doc/html/rfc6750). Para eso, crea una API key y usa su `secret` JWT en el header `Authorization`.

:::danger
Envía todos los requests por HTTPS. Requests por HTTP retornan `403`.

Mantén `secret` y `refresh_token` en secreto.
:::

## Cómo funciona

Sigue este flujo para autenticar cualquier request a la API:

1. Crea una API key para tu servicio
2. Guarda el `secret` JWT que recibes en la creación
3. Envía ese `secret` en el header `Authorization`
4. La API valida el token y resuelve la identidad de la API key
5. Si el token es válido, la request pasa a la capa de autorización

:::info
Autenticación responde la pregunta “quién está llamando”.

Autorización responde la pregunta “qué puede hacer”. Revisa la [guía de autorización](./authorization).
:::

## API keys por entorno

Cada API key pertenece a un entorno:

- Sandbox: prefijo `ak_sandbox_`
- Production: prefijo `ak_live_`

## Header de autenticación

Usa este formato:

```bash
Authorization: Bearer <jwt_secret>
```

`<jwt_secret>` es el campo `secret` que recibes al crear o renovar la API key.

No uses el identificador `ak_sandbox_...` o `ak_live_...` en el header.

## Gestión de API keys

Gestiona API keys desde estos endpoints:

- Crear API key: [`POST /api_keys`](./resources/create-api-key.api.mdx)
- Listar API keys: [`GET /api_keys`](./resources/index-api-keys.api.mdx)
- Ver API key: [`GET /api_keys/{id}`](./resources/get-api-key.api.mdx)
- Actualizar API key: [`PATCH /api_keys/{id}`](./resources/update-api-key.api.mdx)
- Renovar API key: [`PATCH /api_keys/{id}/refresh`](./resources/refresh-api-key.api.mdx)
- Revocar API key: [`DELETE /api_keys/{id}`](./resources/delete-api-key.api.mdx)

## Renovación de API keys

Cuando una API key tiene expiración, la API entrega también un `refresh_token`.

Con ese token puedes renovar la key y obtener:

- Un nuevo `secret`
- Un nuevo `refresh_token`

:::warning[Período de gracia del refresh token]
Después de que expira una API key, su `refresh_token` sigue válido por 60 días.

Si no renuevas dentro de ese período, tendrás que crear una API key nueva.
:::

:::info
`secret` y `refresh_token` solo se muestran en creación y renovación.
:::

## Buenas prácticas

- Crea una API key por servicio o integración
- Usa solo los permisos mínimos que cada servicio necesita
- Separa keys de sandbox y production
- Implementa rotación y revocación de keys
- Guarda `secret` y `refresh_token` en un secret manager
- Renueva antes de expirar para evitar interrupciones
